# Hotel Ryzlink — концепт сайту + AI-консьєрж + адмінка

Концепт для показу замовнику: односторінковий лендінг **Hotel Ryzlink** (Мікулов, Чехія),
відтворений за реальним дизайном оригіналу, з інтегрованим **AI-агентом бронювання** та
**адмін-панеллю**, з якої керують усім контентом і де видно всі переписки й бронювання.

Усе працює на одному **zero-dependency Node-сервері** (`node server.js`, без `npm install`).

## Три частини

### 1. Публічний сайт — `/`
Темна тема оригіналу (графіт + золото + червоний). Секції як на hotelryzlink.cz:
hero-слайдер + панель бронювання → слайдер номерів → галерея-мозаїка (6 категорій) з
лайтбоксом → банери Wine Wellness / Ресторан / Виноробня → слайдер пакетів → контакти → футер.
**81 реальне фото готелю** локально в `public/assets/gallery/`. Без переходів на інші сторінки.

### 2. AI-консьєрж (чат унизу праворуч) — гібрид
- Є `ANTHROPIC_API_KEY` → **справжній Claude** з *tool-use*: реально викликає
  `search_availability`, `create_reservation` тощо і бронює прямо в діалозі.
- Немає ключа → **офлайн-симуляція** на тих самих даних (демо не «падає» на презентації).
- Кожен діалог і бронювання логуються в БД → видно в адмінці.

### 3. Адмінка — `/admin`
Панель керування (пароль з `ADMIN_PASSWORD`, за замовч. `ryzlink-admin`):
- **Дашборд** — статистика (номери, пакети, фото, переписки, бронювання).
- **Номери / Пакети** — CRUD, ціни, фото, вмик/вимк, **порядок = пріоритет** (стрілки ▲▼).
- **Галерея** — категорії та фото (завантаження файлів або за URL).
- **Банери / Контент** — тексти банерів, hero, контакти готелю, коротка FAQ.
- **База знань** — 45+ записів, спарсених з усього сайту (години, ціни, правила, wellness,
  ресторан, дегустації, події, зручності номерів, локація…). **Саме з них AI-агент
  відповідає на будь-яке питання гостя.** CRUD, ключові слова, пошук.
- **AI-агент** — модель, персона/системний промпт, **пріоритети пропозицій**, привітання
  та швидкі кнопки чату (одразу впливають на поведінку агента).
- **Переписки** — усі діалоги з агентом: повні транскрипти, що саме він пропонував, які брони вийшли.
- **Бронювання** — таблиця всіх бронювань (із сайту й з чату).

**Ключова ідея архітектури:** сайт **керується з БД** — сервер віддає `/js/data.js`
динамічно з `data/db.json`, тож будь-яка правка в адмінці одразу видно на сайті (після reload).

## Запуск

Потрібен **Node.js 18+**. Залежностей ставити не треба.

```bash
node server.js
#  Сайт:    http://localhost:3000
#  Адмінка: http://localhost:3000/admin   (пароль: ryzlink-admin)
```

### Налаштування (`.env`, необов'язково)

```bash
cp .env.example .env
```
```
ANTHROPIC_API_KEY=sk-ant-...     # увімкнути справжнього Claude (інакше — симуляція)
MODEL=claude-sonnet-5            # або claude-opus-4-8 (також змінюється в адмінці)
ADMIN_PASSWORD=ваш-пароль        # ОБОВ'ЯЗКОВО змініть для реального показу
PORT=3000
```

## Структура

```
server.js              zero-dependency сервер: сайт + /api + адмін-API + авторизація
db.js                  файлова БД (data/db.json), логи переписок і бронювань
data/db.json           база (створюється авто, сідиться з data.js) — джерело правди
public/
  index.html           лендінг (усі пункти меню — якорі)
  css/styles.css       темна тема оригіналу
  js/data.js           СІД-дані (сервер віддає їх динамічно з БД як /js/data.js)
  js/booking.js        мок системи бронювання (за зразком D-EDGE)
  js/main.js           слайдери, галерея-лайтбокс, модалка бронювання
  js/chat.js           AI-консьєрж (Claude + симуляція) + логування
  assets/gallery/      81 реальне фото готелю
  admin/               адмін-панель (index.html, admin.css, admin.js)
```

## Інфраструктура оригіналу (з дослідження)

- **CMS:** власна, агенція **Newlogic** (не WordPress/Wix).
- **Бронювання:** рушій **D-EDGE Hospitality Solutions** (`secure-hotel-booking.com`), PCI-DSS.
  Для продакшену AI-агент і адмінка інтегрувалися б з **API D-EDGE** замість мока у `booking.js`.

## Примітка

Демонстраційний прототип: реальні платежі не проводяться, бронювання підтверджується
символічним номером. Фото — реальні знімки готелю (для концепту).
